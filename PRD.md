# 日程提醒小程序 — 需求文档

> **当前版本**：v0.1（已测试通过）
> **最后更新**：2026-04-27

---

## 一、产品定位

一款轻量级微信小程序，帮助用户快速创建、查看和管理日常日程，核心路径极简：**打开 → 添加 → 查看**。

---

## 二、已实现功能（v0.1）

### 2.1 添加日程页（pages/index）

| 字段 | 类型 | 是否必填 | 说明 |
|------|------|:--------:|------|
| 日程标题 | 文本输入 | ✅ | 单行，placeholder "请输入日程标题" |
| 日期 | 日期选择器 | ✅ | 默认起始日为当天，不可选过去日期 |
| 详情 | 多行文本 | ❌ | 最多500字，placeholder "请输入日程详情（可选）" |

- 保存按钮：标题+日期都填写后才可点击，否则灰色禁用态
- 保存逻辑：写入 `wx.setStorageSync('schedules')`，以数组形式存储
- 保存成功：Toast 提示"保存成功"，1.5秒后自动返回上一页
- 数据结构：
  ```json
  {
    "id": 1745673600000,
    "title": "开会",
    "date": "2026-04-27",
    "detail": "季度复盘",
    "createdAt": "2026-04-27T14:00:00.000Z"
  }
  ```

### 2.2 日程列表页（pages/list）

- **列表展示**：卡片式布局，显示标题、日期标签、详情（最多2行截断）
- **排序规则**：按日期倒序（最近的在前），同一天按创建时间倒序
- **点击卡片**：弹窗显示完整标题、日期、详情
- **删除**：点击"删除"按钮 → 二次确认弹窗 → 删除并刷新列表
- **空状态**：无日程时显示"📅 还没有日程 / 点击右下角按钮添加"
- **浮动按钮**：右下角"+"，跳转到添加日程页
- **自动刷新**：从添加页返回时 `onShow` 重新加载数据

## 四、数据存储方案（当前）

| 项目 | 当前方案 | 备注 |
|------|---------|------|
| 存储位置 | `wx.setStorageSync('schedules')` | 本地存储，上限 10MB |
| 数据格式 | JSON 数组 | 每条含 id / title / date / detail / createdAt |
| 潜在风险 | 换设备/清缓存数据丢失 | 如需保留，后续需上云 |
