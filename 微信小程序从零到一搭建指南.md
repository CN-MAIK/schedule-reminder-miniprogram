# 微信小程序日程提醒应用 — 从零到一完整搭建指南

> **项目地址**：https://github.com/CN-MAIK/schedule-reminder-miniprogram  
> **版本**：v1.1（含云开发+提醒推送）  
> **适用**：零基础产品经理/开发者学习微信小程序开发全流程

---

## 一、项目概述

### 1.1 这是什么
一个**微信小程序日程提醒应用**，核心功能：
- ✅ 添加日程（标题、日期、时间、详情）
- ✅ 列表展示（全部/待办/已完成筛选）
- ✅ 时间选择器（日期+时间）
- ✅ 提前提醒（5分钟/10分钟/15分钟/30分钟/1小时前）
- ✅ 订阅消息推送（到时间自动微信通知）
- ✅ 编辑/删除日程
- ✅ 已完成标记

### 1.2 技术栈
| 层级 | 技术 |
|------|------|
| 前端 | 微信小程序原生框架（WXML + WXSS + JS） |
| 后端 | 微信云开发（云数据库 + 云函数 + 定时触发器） |
| 存储 | 本地存储(wx.getStorageSync) + 云数据库 |
| 推送 | 微信订阅消息（一次性授权） |

## 二、前置准备

### 2.1 注册微信小程序
1. 访问 https://mp.weixin.qq.com
2. 点击"立即注册" → 选择"小程序"
3. 填写邮箱、密码、主体信息（个人/企业）
4. 完成微信认证
5. 记录 **AppID**（如：`wxf2097a44892239f3`）

### 2.2 下载微信开发者工具
1. 访问 https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html
2. 下载对应系统版本的稳定版
3. 安装并扫码登录

### 2.3 申请订阅消息模板
1. 登录 mp.weixin.qq.com
2. 左侧菜单 → **功能** → **订阅消息**
3. 点击"添加" → 搜索关键词（如"提醒"、"日程"）
4. 选择合适的模板，记录 **模板ID**
5. 查看模板详情，记录字段编号（如 `thing2`、`date4`）

> ⚠️ 订阅消息是**一次性授权**，用户每次需要提醒时都要点击同意。

### 2.4 开通云开发
1. 开发者工具中点击顶部 **"云开发"** 按钮
2. 按提示创建环境（免费版足够）
3. 记录 **环境ID**（如：`cloudbase-d2gpr441j7abd606c`）

### 4.4 页面结构（WXML）

微信小程序页面由三部分组成：
- **.wxml** — 结构（类似HTML）
- **.wxss** — 样式（类似CSS）
- **.js** — 逻辑（JavaScript）

#### 4.4.1 列表页（list.wxml）核心结构

```xml
<view class="container">
  <!-- 筛选标签 -->
  <view class="filter-bar">
    <view class="filter-tab {{filter === 'all' ? 'active' : ''}}" 
          bindtap="onFilter" data-filter="all">全部</view>
    <view class="filter-tab {{filter === 'pending' ? 'active' : ''}}" 
          bindtap="onFilter" data-filter="pending">待办</view>
    <view class="filter-tab {{filter === 'completed' ? 'active' : ''}}" 
          bindtap="onFilter" data-filter="completed">已完成</view>
  </view>

  <!-- 日程列表 -->
  <view class="list">
    <view class="card" wx:for="{{filteredList}}" wx:key="id" 
          bindtap="onItemTap" data-index="{{index}}">
      <view class="check-circle" catchtap="onToggleComplete" data-id="{{item.id}}">
        <text wx:if="{{item.completed}}">✓</text>
      </view>
      <text class="card-title">{{item.title}}</text>
      <text class="card-date">{{item.date}} {{item.time}}</text>
      <text wx:if="{{item.remind}}">🔔</text>
    </view>
  </view>

  <!-- 浮动添加按钮 -->

```

**关键语法**：
- `{{变量}}` — 数据绑定
- `wx:for` — 列表渲染
- `wx:if` — 条件渲染
- `bindtap` — 点击事件（冒泡）
- `catchtap` — 点击事件（阻止冒泡）
- `data-xxx` — 自定义数据属性

### 4.5 页面逻辑（JS）

#### 4.5.1 数据流核心

```javascript
Page({
  data: {
    // 页面响应式数据，修改后自动更新视图
    title: '',
    date: '',
    time: '',
    remind: false,
    schedules: []  // 日程列表
  },

  // 加载时从本地存储读取
  onLoad() {
    const schedules = wx.getStorageSync('schedules') || [];
    this.setData({ schedules });
  },

  // 保存到本地存储
  onSave() {
    const schedules = wx.getStorageSync('schedules') || [];
    schedules.push({
      id: Date.now(),
      title: this.data.title,
      date: this.data.date,
      time: this.data.time,
      remind: this.data.remind
    });
    wx.setStorageSync('schedules', schedules);
  }
});
```

#### 4.5.2 订阅消息授权

```javascript
requestSubscribe() {
  const templateId = '2ntpB1-KftDWjdhkLA1tWUhHWjJc1Xfv1gleKt0X0nY';
  wx.requestSubscribeMessage({
    tmplIds: [templateId],
    success(res) {
      // res[templateId] === 'accept' 表示用户同意
      return res[templateId] === 'accept';
    }
  });
}
```

#### 4.5.3 云数据库同步

```javascript
async syncToCloud(schedule) {
  const db = wx.cloud.database();
  const collection = db.collection('schedules');
  
  // 新增记录
  const res = await collection.add({
    data: {
      ...schedule,
      reminded: false  // 是否已发送提醒
    }
  });
  return res._id;  // 云数据库记录ID
}
```

## 五、云开发详解

### 5.1 云数据库

**集合（Collection）**：`schedules`

**数据结构设计**：
```json
{
  "_id": "自动生成的记录ID",
  "_openid": "用户唯一标识（自动填充）",
  "id": 1745673600000,
  "title": "开会",
  "date": "2026-04-27",
  "time": "14:00",
  "detail": "项目进度汇报",
  "remind": true,
  "remindIndex": 2,
  "remindAt": "2026-04-27T06:45:00.000Z",
  "reminded": false,
  "subscribed": true,
  "completed": false,
  "createdAt": "2026-04-27T06:00:00.000Z",
  "updatedAt": "2026-04-27T06:00:00.000Z"
}
```

| 字段 | 说明 |
|------|------|
| `_id` | 云数据库自动生成的主键 |
| `_openid` | 用户OpenID，云函数自动填充 |
| `id` | 本地存储用的自定义ID |
| `remindAt` | 实际提醒触发时间（ISO格式） |
| `reminded` | 是否已推送过（防重复） |
| `subscribed` | 用户是否授权了订阅消息 |

**权限设置**：仅创建者可读写（每个用户只能看到自己的数据）

### 5.2 云函数（remind）

**功能**：每分钟检查一次，给到期日程发送订阅消息

**触发方式**：定时触发器（Cron表达式）

**config.json**：
```json
{
  "triggers": [
    {
      "name": "remindTimer",
      "type": "timer",
      "config": "0 * * * * * *"
    }
  ]
}
```

**Cron表达式**：`0 * * * * * *`
- 格式：`秒 分 时 日 月 星期 年`
- `0 * * * * * *` = 每分钟的第0秒执行一次

**核心逻辑**：
```javascript
exports.main = async (event, context) => {
  const now = new Date();
  const windowEnd = new Date(now.getTime() + 60 * 1000);

  // 查询：remind=true + 未提醒 + remindAt在当前1分钟窗口内
  const res = await db.collection('schedules')
    .where({
      remind: true,
      reminded: _.neq(true),
      remindAt: _.gte(now.toISOString()).and(_.lt(windowEnd.toISOString()))
    })
    .get();

  // 逐条发送订阅消息
  for (const item of res.data) {
    await cloud.openapi.subscribeMessage.send({
      touser: item._openid,
      templateId: TEMPLATE_ID,
      page: `pages/detail/detail?id=${item.id}`,
      data: {
        thing2: { value: item.title.substring(0, 20) },
        date4: { value: `${item.date} ${item.time}` }
      }
    });
    
    // 标记已提醒
    await db.collection('schedules').doc(item._id).update({
      data: { reminded: true }
    });
  }
};
```

## 七、关键踩坑记录

### 坑1：Git架构不匹配
**现象**：`git.exe` 运行报错，提示架构不匹配  
**原因**：安装了 ARM64 版本的 Git，但电脑是 Intel x64  
**解决**：卸载旧版，从 git-scm.com 下载 x64 版本重新安装

### 坑2：SSL证书验证失败
**现象**：`UNABLE_TO_VERIFY_LEAF_SIGNATURE`  
**原因**：Node.js 证书链问题  
**解决**：临时设置环境变量 `NODE_TLS_REJECT_UNAUTHORIZED=0`

### 坑3：云函数目录结构
**现象**：开发者工具左侧看不到 cloudfunctions 文件夹  
**原因**：小程序代码和云函数混在同一目录，没有正确设置 miniprogramRoot  
**解决**：重构目录，小程序代码移入 `src/`，云函数留在根级 `cloudfunctions/`

### 坑4：订阅消息模板字段
**现象**：推送成功但用户收不到消息  
**原因**：云函数中字段名（thing1/time2）与实际模板字段（thing2/date4）不匹配  
**解决**：登录 mp.weixin.qq.com 查看模板详情，修改云函数中的字段映射

### 坑5：定时触发器未上传
**现象**：云函数部署了但从不自动执行  
**原因**：只部署了代码，没有上传触发器配置  
**解决**：右键云函数文件夹 → "上传触发器"
